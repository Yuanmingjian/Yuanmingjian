答 在 JavaScript 中，super 关键字主要用于访问对象的原型对象中的属性或方法，而不是用于访问父类的属性或方法。在原型链中，一个对象的原型对象并不一定是它的父类，而是它所继承的对象。

当我们使用 Object.setPrototypeOf(obj2, obj1) 将 obj1 设置为 obj2 的原型时，obj2 实际上是继承了 obj1 对象的属性和方法，而不是从一个类继承而来。因此，在这个情况下，super 所指向的是 obj1 这个原型对象，而不是父类。

在 JavaScript 中，类（class）和原型（prototype）是两种不同的面向对象编程的机制。在 ES6 中引入了类的概念，但本质上 JavaScript 还是基于原型继承的。所以在这个上下文中，super 关键字指向对象的原型，而不是类的父类。 super指向父类
